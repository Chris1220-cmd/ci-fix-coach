# Contributing to CI Fix Coach

Thank you for your interest in contributing! CI Fix Coach is a GitHub Action that automatically diagnoses CI failures using Claude AI. Contributions of all kinds are welcome.

## Ways to Contribute

- **Bug reports** — Open an issue if something doesn't work as expected
- - **Feature requests** — Suggest new CI failure categories or output improvements
  - - **Code contributions** — Fix bugs or add features via pull requests
    - - **Documentation** — Improve the README, add examples, or fix typos
      - - **Testing** — Try CI Fix Coach on your own repos and share feedback
       
        - ## Good First Issues
       
        - Look for issues labeled [`good first issue`](https://github.com/Chris1220-cmd/ci-fix-coach/labels/good%20first%20issue) — these are well-scoped tasks suitable for first-time contributors.
       
        - ## Development Setup
       
        - ```bash
          # Clone the repo
          git clone https://github.com/Chris1220-cmd/ci-fix-coach.git
          cd ci-fix-coach

          # Install dependencies
          npm install

          # Run tests
          npm test

          # Build (bundles into dist/ with ncc)
          npm run build
          ```

          ## Project Structure

          ```
          src/           # Main source code (index.js)
          tests/         # Test suite
          dist/          # Bundled output (generated by npm run build)
          .github/       # Workflow files
          action.yml     # GitHub Action definition
          ```

          ## Submitting a Pull Request

          1. Fork the repository
          2. 2. Create a branch: `git checkout -b my-feature`
             3. 3. Make your changes and run `npm test`
                4. 4. Run `npm run build` to update `dist/`
                   5. 5. Commit and push your branch
                      6. 6. Open a pull request with a clear description of the change
                        
                         7. ## Reporting Bugs
                        
                         8. When opening a bug report, please include:
                         9. - What CI runner / OS you're using
                            - - The workflow file snippet (with secrets redacted)
                              - - The actual vs expected behavior
                                - - Relevant logs (with sensitive data removed)
                                 
                                  - ## Code of Conduct
                                 
                                  - Be kind and constructive. We're all here to make CI less painful.
